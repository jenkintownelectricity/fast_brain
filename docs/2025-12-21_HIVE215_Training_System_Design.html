üéØ Optimal Hyperparameters

Unsloth-recommended defaults for QLoRA

learning_rate: 2e-4 lora_r: 16 lora_alpha: 16 batch_size: 2 gradient_accumulation: 4 epochs: 3-5

üìä Data Quality Targets

Minimum requirements for good results

Examples: 50-100 (minimum 10) Avg Tokens: 200-500 Topic Coverage: 10+ topics Quality: Expert-level responses

üíæ Memory Optimization

When hitting CUDA OOM errors

1. Reduce batch_size ‚Üí 1 2. Enable gradient_checkpointing 3. Lower lora_r ‚Üí 8 4. Use 3B model instead of 8B

üìà Loss Interpretation

What loss values mean

> 3.0: Not learning (check data) 2.0-3.0: Learning, needs more 1.5-2.0: Good progress < 1.5: Excellent (watch overfit)
